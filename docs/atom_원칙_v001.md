# Atom 원칙 v001

작성일: 2026-06-18

## 0. 업데이트 방식

이 문서는 법원직.com 및 관련 시험 데이터의 atom 제작 기준을 누적 관리하기 위한 기준 문서이다.

- 새 원칙이 확정되면 이 문서에 누적 반영한다.
- 기준이 크게 바뀌면 `atom_원칙_v002.md`처럼 새 버전을 만들고, 이전 버전은 보존한다.
- 실제 작업 중 판단이 흔들리면 이 문서의 원칙을 우선한다.
- 단순 보기 변환 규칙보다, 문제를 해결하게 하는 조문·판례·학설의 근거 지점을 우선한다.

## 1. Atom의 정의

Atom은 기출 지문을 O/X로 판단하게 만드는 최소 법리 명제이다.

Atom은 문제 선지의 문장을 그대로 옮긴 것이 아니다. 각 지문이 O 또는 X가 되는 근거인 조문, 판례, 학설의 판단 지점을 찾아서, 그 지점을 독립적인 법리 문장으로 만든 것이다.

좋은 atom은 다음 요건을 모두 만족해야 한다.

1. 자기완결식 문장이다.
2. 중립적인 평서문이다.
3. 하나의 법리 또는 하나의 판단 지점만 담는다.
4. 다른 문제에서도 재사용할 수 있는 법리 단위이다.
5. 조문, 판례, 학설 중 적어도 하나의 근거를 가진다.

## 2. 원문 보존 원칙

문제 원문은 atom으로 덮어쓰지 않는다.

다음 정보는 출처 확인용으로 그대로 보존한다.

- 시험명
- 연도와 회차
- 과목
- 문항번호
- 문제 본문
- 보기 ①~⑤
- 박스 지문 ㄱ, ㄴ, ㄷ, ㄹ 등
- 공식 정답
- 출제 당시 정답
- 현재 기준 정답 또는 현재 기준 수정 필요 여부

원문은 원문이고, atom은 원문을 풀기 위한 법리 데이터이다.

## 3. 모든 지문 분해 원칙

일반 보기뿐 아니라 개수형, 조합형, 박스형도 모두 atom 추출 대상이다.

- ①~⑤ 보기형은 각 보기를 판단 단위로 분해한다.
- 박스형은 ㄱ, ㄴ, ㄷ, ㄹ 등 각 박스 지문을 판단 단위로 분해한다.
- 조합형은 각 조합 자체가 아니라 조합을 구성하는 개별 지문을 판단 단위로 분해한다.
- 개수형도 개수를 맞히는 문제로 끝내지 않고, 개수를 구성하는 각 지문의 O/X 근거를 모두 찾아 atom화한다.

공식 정답만으로 개별 지문의 O/X가 바로 확정되지 않는 경우에는 조문, 판례, 학설, 해설을 통해 각 지문을 별도로 검증한다.

## 4. 근거 지점 atom화 원칙

Atom은 보기 문장 압축본이 아니라, O/X 판단의 근거 지점이다.

근거 유형은 다음과 같이 구분한다.

- 조문: 법률, 규칙, 명령, 부칙 등 성문 규정의 요건·효과·기간·절차
- 판례: 대법원, 헌법재판소 등 판례가 제시한 법리, 기준, 결론
- 학설: 통설, 다수설, 판례와 대립되는 학설, 시험상 요구되는 이론 구조

사례형 지문은 사실관계를 그대로 atom으로 만들지 않는다. 그 사례를 해결하는 조문 요건, 판례 법리, 학설상 판단 기준을 atom으로 만든다.

## 5. O atom 원칙

대표 atom은 원칙적으로 O 문장이어야 한다.

O atom은 현재 기준으로 정확한 법리 문장이어야 한다. 출제 당시에는 맞았지만 현재 법개정 또는 판례변경으로 결론이 달라진 경우에는 다음을 함께 보존한다.

- 원문 지문
- 출제 당시 판단
- 현재 기준 판단
- 현재 기준 수정 atom
- 검증일자
- 변경 근거

## 6. X atom 종속 원칙

X 지문은 틀린 문장 자체를 독립 지식처럼 만들지 않는다.

틀린 지문은 원문으로 보존하되, atom 구조에서는 올바른 O atom 또는 올바른 근거 법리에 종속시킨다.

X 지문을 만들거나 보존할 때는 다음 원칙을 따른다.

- X는 O atom의 핵심 한 부분만 바뀐 형태여야 한다.
- X가 왜 틀렸는지 설명하는 `trap` 또는 `error_point`를 둔다.
- X에는 대응되는 올바른 문장 또는 대표 O atom을 연결한다.
- 하나의 X 문장 안에 여러 함정을 섞지 않는다.

## 7. 자기완결식 문장 원칙

Atom은 단독으로 읽어도 의미가 완결되어야 한다.

다음 표현은 atom에 남기지 않는다.

- 위 사례
- 위 ③번
- 이 경우
- 다음 중
- 이에 대한 설명
- 앞의 사안
- 보기와 같은 경우
- 옳은 것은
- 옳지 않은 것은
- 물음표로 끝나는 의문문
- 키워드만 나열한 명사구

사례 인물인 갑, 을, 병, 정, A, B, C 등도 원칙적으로 제거한다. 다만 판례 법리의 추상화에 필요한 지위명은 채권자, 채무자, 매수인, 임차인, 피고인처럼 일반 법률관계명으로 바꾼다.

## 8. 문장 품질 원칙

Atom 문장은 학습 데이터로 바로 노출될 수 있어야 한다.

- 띄어쓰기가 깨진 문장은 수정한다.
- OCR 오류와 특수문자 오류를 고친다.
- 불필요한 괄호, 번호, 보기 기호를 제거한다.
- 한 문장에 여러 법리를 억지로 붙이지 않는다.
- 장황한 사례 설명 대신 판단 기준을 쓴다.
- 문제풀이식 표현 대신 법리식 표현을 쓴다.

나쁜 예:

> 위③번의 경우 특별한 사정이 없는 한 공로로 제공된 도로 지하부분에 매설된 시설에 대한 철거 등을 구하는 것은 권리남용에 해당하지 않는다.

수정 방향:

> 토지소유자가 토지를 일반 공중의 통행에 제공하여 독점적·배타적 사용수익권 행사가 제한되는 경우에도, 특별한 사정이 있으면 지하 매설 시설의 철거청구가 권리남용으로 제한될 수 있다.

## 9. 검증 정보 원칙

각 atom에는 최소한 다음 검증 정보를 붙인다.

- 검증일자
- 근거유형: 조문, 판례, 학설
- 근거출처: 조문번호, 판례번호, 판결일자, 학설명 등
- 원문 출처: 시험명, 연도, 과목, 문항, 보기 또는 박스 지문 번호
- 출제 당시 판단
- 현재 기준 판단
- 현재 기준 수정 여부

법개정 여부를 임의로 추측해서 문제 삼지 않는다. 현재 기준 판단이 불명확하거나 조문·판례 확인이 필요한 경우에는 판단 보류 상태로 표시하고, 확인 가능한 근거를 찾은 뒤 반영한다.

## 10. 회차별 atom 생성 원칙

각 회차는 먼저 독립 atom 파일로 만든다.

예:

- `2025_법무사_민법_atom.json`
- `2025_법무사_헌법_atom.json`
- `2024_법무사_민법_atom.json`

회차별 atom에는 해당 회차 문제를 풀기 위한 근거 atom과 원문 연결 정보를 모두 담는다.

## 11. 통합 atom 생성 원칙

통합 atom은 같은 조문, 판례, 학설 지점을 묻는 atom을 병합한 것이다.

병합 기준은 문장 유사도가 아니라 법리 동일성이다.

다음이 같거나 실질적으로 같으면 하나의 통합 atom으로 묶는다.

- 같은 조문 요건
- 같은 조문 효과
- 같은 판례 법리
- 같은 학설상 결론
- 같은 예외 또는 제한 법리

통합 atom에는 출제 이력을 누적한다.

- 시험명
- 연도
- 회차
- 과목
- 문항번호
- 보기번호 또는 박스 지문번호
- 원문 지문
- 당시 O/X
- 현재 기준 O/X

## 12. 다른 시험과 통합 원칙

법원직, 변호사시험, 법무사시험 등 다른 시험의 atom을 통합할 때도 기준은 법리 동일성이다.

원문이 달라도 같은 조문·판례·학설 지점을 묻는다면 하나의 atom으로 병합한다. 다만 출처와 원문은 각각 보존한다.

서비스 화면에는 필요한 atom과 근거 정보를 제공하고, 문제 원문 전체 데이터는 로컬 private 또는 Supabase 보안 구조에 맞춰 관리한다.

## 13. 최종 검수 질문

atom 하나를 만들 때마다 다음 질문을 확인한다.

1. 이 atom은 문제 선지 복사가 아니라 O/X 판단의 근거 법리인가?
2. 조문, 판례, 학설 중 근거가 있는가?
3. 단독으로 읽어도 완결되는가?
4. 중립적 평서문인가?
5. 한 문장에 하나의 판단 지점만 담았는가?
6. 사례 인물, 보기번호, 지시어가 제거되었는가?
7. X 지문은 올바른 O atom에 종속되어 있는가?
8. 출제 당시 판단과 현재 기준 판단이 구분되어 있는가?
9. 검증일자가 있는가?
10. 같은 atom을 다른 회차나 다른 시험에도 재사용할 수 있는가?
